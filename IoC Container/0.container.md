### ğŸ”¹ Basic Meaning

A **container** is a **manager**.

> In software, a **container** is a system that **creates, holds, manages, and controls objects or components** for you.

Instead of you doing everything manually, the **container does it automatically**.

## ğŸŒ± Container in **Spring Framework**

In Spring, **Container = IoC Container**

It:

* Creates **Beans**
* Injects dependencies
* Manages lifecycle
* Handles scope (singleton, prototype, etc.)

Example:

```java
@Component
class UserService {}
```

Spring container:

* Detects this class
* Creates object
* Stores it
* Reuses it when needed
